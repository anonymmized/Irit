# Security Policy

## Supported scope

Security-sensitive areas include:

- SSH authentication flow
- sudo escalation flow
- remote helper generation
- client export bundle generation
- rollback and checkpoint handling

## Reporting a vulnerability

Please do not open public issues for security problems.

Report privately to:

- GitHub Security Advisory, if enabled
- or the maintainer email configured for the repository

Include:

- affected version
- reproduction steps
- expected impact
- logs or screenshots if relevant

## Response goals

- acknowledge the report quickly
- confirm severity
- prepare a fix or mitigation
- publish a documented patch
